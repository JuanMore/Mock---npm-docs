Features

npm is the world's largest software registry

Easy access

Open source developers from every continent use npm to share and borrow packages, and many organizations use npm to manage private development as well.

Unscoped public packages exist in the global public registry namespace and can be referenced in a package.json file with the package name alone: package-name. Scoped public packages belong to a user or organization and must be preceded by the user or organization name when included as a dependency in a package.json file: @username/package-name @org-name/package-name

With npm private packages, you can use the npm registry to host code that is only visible to you and chosen collaborators, allowing you to manage and use private code alongside public code in your projects.

npm consists of three distinct components:
the website
the Command Line Interface (CLI)
the registry

npm is the package manager for the Node JavaScript platform. It puts modules in place so that node can find them, and manages dependency conflicts intelligently.

Most commonly, you use it to publish, discover, install, and develop node programs.Run npm help to get a list of available commands
